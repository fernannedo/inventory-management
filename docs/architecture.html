Filter Data Flow

1. User input
User selects a filter in FilterBar.vue — e.g. Location = “San Francisco”, Category = “Sensors”, Period = “2025-01”, Status = “Delivered”
↓
2. Reactive state
useFilters.js composable updates shared refs: selectedLocation.value = "San Francisco". All views sharing the composable observe the change instantly.
↓
3. Watch fires
Each view has watch([selectedPeriod, selectedLocation, selectedCategory, selectedStatus], loadData). The watcher calls loadData() with getCurrentFilters().
↓
4. HTTP request
api.js builds URLSearchParams and fires:
GET /api/orders?warehouse=San+Francisco&category=sensors&status=delivered&month=2025-01
↓
5. Backend filters
main.py applies sequential in-memory filters:
apply_filters(orders, warehouse, category, status) → filter_by_month(result, month). Quarter strings (e.g. Q1-2025) expand to 3 month values via QUARTER_MAP.
↓
6. JSON response
FastAPI serialises the filtered list through the Pydantic Order model and returns a JSON array. Pydantic strips any extra fields and validates types.
↓
7. Re-render
The view sets orders.value = response.data. All computed properties recalculate (KPIs, charts, aggregates) and Vue re-renders the template.

Key Design Patterns

Singleton Composables

useFilters() returns the same refs across all components — filter state is globally shared without a store.

Refs vs Computed

Mutable state lives in ref() (raw data, filter values). All derived values are computed() and recalculate automatically.

Watch-Driven Fetching

Each view watches all 4 filter refs. Any change triggers loadData(), keeping the API call co-located with data ownership.

Sequential In-Memory Filtering

Backend applies filters one at a time — warehouse, then category, then status, then month. Never mutates the original in-memory list.

Pydantic Response Models

Every endpoint declares response_model=List[X]. Pydantic validates output, strips unexpected fields, and generates OpenAPI docs automatically.

Props Down / Events Up

Modals receive data via props (:product) and close via emits (@close). No shared state needed for modal interactions.

SVG/CSS Charts

All charts (donut, bar, stacked bar) are built with raw SVG and CSS flex. No chart library dependency.

i18n + Currency

Locale toggle (en/ja) stored in localStorage. Currency follows locale: USD for English, JPY for Japanese (1 USD = 150 JPY).
